In this sketch, I played around with the different transform tools, such as translate, rotate, push, pop, etc., as well as using speedX and speedY variables for the bounce.

Exercise 4

This exercise is a continuation of Exercise 3. This time, my rabbit characters are part of a class object that bounces back and forth on the canvas screen using the position and speed variables for x and y, known as ( posx, posy, speedx[sx], speedy[sy] ). More importantly, I've learned how to convert my rabbit character into a class object by introducing variables, the use of this in variables, and taking away the word function for functions inside class objects.

Project 2

For this project, I'm trying to create a spiral that starts off as a tiny dot, then grows and rotates clockwise as the transformation progresses. Then, it flies off to the top left corner of the screen and comes back to the center of the screen as a normal spirograph. A second spirograph is created that goes twice the speed and a third one spreads through the screen and exits at the right side of the screen. The most difficult part about this project is not getting the push/pop function to work on spirographs since the shape is traced and leaves a permanent path, compared to simple, static shapes where it can be easily transformed over time. The color theme for this project is black and white.
Original algorithm link: https://p5js.org/examples/simulate-spirograph.html
Feedback from Steve Durie: Lower frame rate to 2 frames per second and add a third element, extend animation to 25-30 seconds (10 seconds more).
